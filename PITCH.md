# 🎯 甩锅分析器 — Pitch Speech

---

## 😤 痛点：职场「背锅侠」之痛

> "谁来说一下为什么上不了线？"
> "前端联调慢了，接口我周三就给了。"

你有没有经历过这样的时刻？明明不是你的问题，但在复盘会上锅就飞过来了。想反驳，但——

- 聊天记录太长，翻不到关键证据
- 时间线混乱，说不清「到底是谁 block 了谁」
- 直接怼回去显得不专业，不怼又憋屈
- 想找经验参考，没有现成的话术模板

**核心矛盾：信息不对称 × 表达能力不对等 = 老实人永远背锅。**

---

## 💡 产品：粘贴聊天记录，AI 帮你反杀

> 粘贴一段聊天记录，AI 自动找出谁该背锅、整理证据链、生成优雅反击话术，还能多轮追问深挖。

### 核心能力一览

| 能力 | 说明 | 价值 |
|------|------|------|
| 🔑 **关键消息速览** | 识别最致命的甩锅消息，严重程度排序，关键词高亮 | 3 秒定位「锅」在哪 |
| 🎯 **锅归属判定** | 量化各方责任占比，饼图 + 条形图可视化 | 谁背锅，用数据说话 |
| 📋 **证据链生成** | 提取时间线，标注延误/推诿/甩锅类型，建立因果链 | 完整还原事情经过 |
| 🗡️ **反击话术** | 三种风格（高情商 / 阴阳怪气 / 对事不对人），引用原话 | 优雅回怼，一键复制 |
| 🤖 **AI 深度分析** | 通义千问大模型，语义级深度分析 + 流式 SSE 输出 | 比关键词更懂人话 |
| 💬 **多轮追问** | 分析完可继续追问 AI，保持上下文连贯 | 越问越深，不留死角 |
| 📝 **社区话术库** | 浏览/贡献/点赞/收藏经典甩锅应对话术 | 站在前人肩膀上反击 |
| 📊 **数据看板** | 个人统计 + 使用趋势 + 分类分布 | 量化你的「反甩锅」成长 |

### 为什么值得做？

1. **刚需** — 每个职场人都被甩过锅
2. **高频** — 跨团队协作的扯皮每天都在发生
3. **AI 天然擅长** — 时间线分析、逻辑推理、文本生成，全是大模型强项
4. **情绪价值** — 用户用完会觉得「终于有人帮我说话了」
5. **社区效应** — 话术库让用户之间互相赋能，形成飞轮
6. **商业闭环** — 免费+Pro+团队版三级套餐，增长路径清晰

## 🛠️ 技术方案

### 产品体验流程

```
粘贴聊天记录 / 截图 OCR 识别
       ↓
┌─────────────────────────────────────────────────────────┐
│  本地快速分析（零延迟、零流量）                              │
│  Step 1  结构化解析 → 兼容钉钉/飞书/微信/邮件/会议纪要      │
│  Step 2  关键词信号检测 → 六维信号（延误/承诺/推诿/甩锅/不满）│
│  Step 3  量化归责 + 模板话术 → 饼图条形图 + 多风格反击        │
└─────────────────────────────────────────────────────────┘
                        OR
┌─────────────────────────────────────────────────────────┐
│  AI 深度分析（通义千问大模型 · SSE 流式输出）                 │
│  Step 1  语义理解 → 理解潜台词、隐含推诿、承诺未兑现          │
│  Step 2  因果推理 → 分析「谁 block 了谁」的依赖链             │
│  Step 3  AI 生成 → 量化归责 + 有理有据的定制化反击话术         │
└─────────────────────────────────────────────────────────┘
       ↓
  输出结果 → 自动保存历史 → 导出 PDF/PNG → 继续追问 AI
       ↓
  话术库浏览 → 数据看板 → 升级套餐解锁更多能力
```

### 产品亮点

| 亮点 | 说明 |
|------|------|
| 🎨 **精致暗色 UI** | 玻璃拟态 + 流光按钮 + 渐变配色，视觉体验拉满 |
| 🤖 **双模式分析** | 本地快速分析（瞬间出结果）+ AI 深度分析（大模型语义理解） |
| 📷 **OCR 识别输入** | 上传聊天截图，qwen-vl 多模态模型自动提取文字 |
| 💬 **多轮追问** | 分析完不够？继续追问 AI，基于上下文深入挖掘 |
| 📚 **历史回顾** | 分析结果自动保存，随时翻阅过往「战绩」 |
| 📤 **一键导出** | 分析报告导出为 PDF / PNG，留档存证 |
| 📝 **社区话术库** | 用户贡献 + 点赞 + 收藏，经典话术共享 |
| 📊 **数据看板** | 使用趋势、分类分布、连续天数，Chart.js 可视化 |
| 💎 **订阅体系** | 免费/Pro/团队三级套餐，按需升级 |
| 🛡️ **安全合规** | API Key 后端托管、JWT 鉴权、按套餐差异化限流 |
| 📱 **全端适配** | 手机 / 平板 / 桌面响应式布局 |

### 技术栈

| 层级 | 选型 |
|------|------|
| 前端 | 原生 JS (ES Module) + CSS 变量 + 响应式 |
| 后端 | TypeScript + Hono (14KB) + Node.js |
| 数据库 | SQLite (@libsql/client) + Drizzle ORM |
| AI | 阿里云百炼 DashScope (qwen-plus / qwen-vl) |
| 认证 | JWT (jose) + bcryptjs |
| 图表 | Chart.js |
| 导出 | html2canvas + jsPDF |

## 🏗️ 系统架构

```
blame-analyzer-V3/
├── server/                        # TypeScript 后端 (Hono + SQLite)
│   ├── src/
│   │   ├── index.ts               # 入口 (Hono + CORS + 路由注册)
│   │   ├── config.ts              # 配置 (含套餐定价)
│   │   ├── routes/                # 8 个路由模块
│   │   │   ├── ai.ts              # AI 分析 (流式+非流式+自动保存)
│   │   │   ├── auth.ts            # 注册/登录/JWT
│   │   │   ├── history.ts         # 历史 CRUD
│   │   │   ├── ocr.ts             # OCR 图片识别
│   │   │   ├── subscription.ts    # 订阅管理
│   │   │   ├── conversation.ts    # 多轮对话
│   │   │   ├── phrases.ts         # 社区话术库
│   │   │   └── dashboard.ts       # 数据看板
│   │   ├── middleware/            # 鉴权 + 限流
│   │   ├── services/              # DashScope + 用户 + 订阅
│   │   └── db/                    # Drizzle ORM + libsql
│   └── package.json
├── js/                            # 前端 ES Module (15 个模块)
│   ├── app.js                     # 主入口
│   ├── ai.js / auth.js / history.js / export.js / ocr.js
│   ├── subscription.js / conversation.js / phrases.js / dashboard.js
│   ├── parser.js / analyzer.js / renderer.js / samples.js / utils.js
├── css/style.css                  # 全局样式 (含 Phase 3 全部组件)
└── index.html                     # 入口页面
```

**设计原则：**
- **前后端分离** — 前端原生 JS SPA + TypeScript API 后端
- **关注点分离** — 15 个前端模块各司其职，8 个后端路由职责清晰
- **安全优先** — API Key 后端托管，JWT 鉴权，密码 bcrypt 加密
- **渐进增强** — 无网络时本地分析可用，AI / 社区 / 看板是增值
- **商业就绪** — 完整订阅体系 + 限流引导 + 套餐差异化
